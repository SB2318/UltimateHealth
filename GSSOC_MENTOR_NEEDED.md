# ✅ Mentor Onboarding & Review Checklist

All mentors should align with the following standards before reviewing PRs.

## Branch Rules

- [ ] All web-related changes must target the **`web` branch**
- [ ] Branch names must follow:

```bash
feat/web-*
fix/web-*
refactor/web-*
```

---

## TypeScript Standards

- [ ] No usage of `any`
- [ ] Strict type safety must be preserved

---

## UI Standards

- [ ] Premium design quality
- [ ] Proper gradients and glassmorphism
- [ ] Dark/light contrast validation
- [ ] Responsive behavior verified

---

## Content Rules

- [ ] No lorem ipsum
- [ ] No placeholder assets
- [ ] No dummy content merged into production

---

## Commit Standards

Use conventional commits:

```bash
feat(web): add reusable modal component
fix(web): resolve navbar overflow bug
refactor(web): optimize auth routing
```
