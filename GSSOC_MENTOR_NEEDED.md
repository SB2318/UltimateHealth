## 4. Mentor Onboarding & Review Guidelines Checklist

When you onboard a mentor, direct them to this checklist to align on code standards:

- [ ] **Branch Checks:** All web changes must target the `web` branch. Branch names must start with `feat/web-`, `fix/web-`, or `refactor/web-`.
- [ ] **No `any`:** Reject PRs that bypass TypeScript safety with `any`.
- [ ] **Premium UI:** Verify that designs adhere to the CSS standards (sleek gradients, glassmorphism, light/dark responsive contrast).
- [ ] **No Placeholders:** Ensure no hardcoded lorem ipsum or dummy assets are merged.
- [ ] **Commit Messages:** Enforce conventional commits e.g. `feat(web): ...` or `fix(web): ...`.
